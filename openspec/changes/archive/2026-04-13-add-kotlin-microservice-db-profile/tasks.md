## 4. Sample entity, repository, migration

- [x] 4.1 **Open question resolved during apply — filename-level conditionals are broken in boilr.** Empirical test: when a file path like `persistence/{{if eq stack_profile "relational-db"}}SampleEntity.kt{{end}}` is rendered with `stack_profile=default`, boilr does NOT skip the file — it writes the file content to a file NAMED after the parent directory (`persistence` as a file, not a directory), which causes collisions when the directory contains multiple conditionally-named files (the second one silently overwrites the first). Switched to **content-level conditionals**: each file has an unconditional name, and its entire body is wrapped in `{{- if eq stack_profile "relational-db" -}}...{{- end -}}`. Under `default`, boilr renders the empty-trimmed output and writes **no file at all** — leaving only empty `persistence/`, `db/migration/` parent directories. Cleaner than expected.
